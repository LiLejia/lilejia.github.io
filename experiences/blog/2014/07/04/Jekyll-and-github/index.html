用Jekyll与Github.io来搭建个人博客

下班，折腾折腾，九点钟边看着一百七十多分钟的《音乐之声》边开始了博客的迁移，现在真的没有比Jekyl+github.io更方便的了吧，Wordpress从来没有觉得顺畅过，当然，最关键的是github.io不需要和Linux打交道，纯静态页面。

现在开始：

没有gem的先安装gem，gem怎么安装的在这里。

三条命令你就能看到你本地的Jekyllrb了：

~ $ gem install jekyll
~ $ jekyll new myblog
~ $ cd myblog
~/myblog $ jekyll serve
# => Now browse to http://localhost:4000

在你自定义的myblog文件夹里，目录应该差不多类似下面：

├── _config.yml
├── _drafts
|   ├── begin-with-the-crazy-ideas.textile
|   └── on-simplicity-in-technology.markdown
├── _includes
|   ├── footer.html
|   └── header.html
├── _layouts
|   ├── default.html
|   └── post.html
├── _posts
|   ├── 2007-10-29-why-every-programmer-should-play-nethack.textile
|   └── 2009-04-26-barcamp-boston-4-roundup.textile
├── _data
|   └── members.yml
├── _site
└── index.html

这些文件与文件夹分别代表的是：

文件/文件夹 内容
_config.yml 包含所有的配置数据，许多选项都可以执行命令行来设置，不过在这个文件里设置能让你少记一些东西
_drafts 文件夹里包含还没有发布的博文，这些文件的名字一般都没有包含日期：title.MARKUP
_includes 包含页面共用的头首侧边栏什么的
_layouts 博文页面的模板
_posts 这些就是你的博文啦
_data 对于Ruby来说，一些固定格式的网站数据可以放到这，jekyll引擎将会自动加载这个文件夹里的.yml和.yaml文件，如果文件名是members.yml，那么你可以通过site.data.members来访问这个数据
_site 这个是生成的站点文件防止的地方，这个文件夹刚好也可以是你的git文件夹
index.html或者其他 都会被Jekyll转换
其他文件与文件夹 其他的一些CSS文件或者图片文件都会直接复制过去。

另外在配置文件里还有很多选项可以设置，不过暂时还没有关心的必要。

接下来是比较关乎日常写博客的内容，我们从Front-matter开始，任何一个有如下YAML Front-matter头格式的文件，将会被Jekyll做特殊处理: --- layout: post title: Blogging Like a Hacker ---

以下是你会常用到的Front-matter选项：

变量 描述
layout 就是你在_layout里面有设置的排版，自带的有default, post, page
permalink 如果你不希望博文的链接是/year/month/day/title.html/你可以主动设置博文的永久链接
published 如果你在生成站点的时候不想发表这篇文章，你可以把这个属性设置成false
category categories 相当于文章的tag、分类或者文件夹
tags 其实都差不多
