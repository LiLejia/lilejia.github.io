如何在不改变Xcode工程的情况下集成Reveal

October 26, 2014

读这篇文章的时候想必你早就非常喜欢用Reveal了，但是当你新建了一个工程而又想快速使用Reveal Debug的时候，你如何快速集成Reveal呢？(或者你不想影响你的原工程不至于在提交的时候忘记移除)。

LLDB 调试器将是你的解决方案

LLDB调试器能够执行你应用内的任意代码，所以你可以直接将Reveal加载到你现在的进程里。

暂停运行iOS，然后在LLDB调试器里输入：

expr (void*)dlopen("/Applications/Reveal.app/Contents/SharedSupport/iOS-Libraries/libReveal.dylib", 0x2);
expr [(NSNotificationCenter*)[NSNotificationCenter defaultCenter] postNotificationName:@"IBARevealRequestStart" object:nil];
continue

十分简单对吧？

我们还可以添加一些例如reveal_load的简单命令来让调试更加简单，你只要新建一个文本文件输入：

command alias reveal_load_sim expr (void*)dlopen("/Applications/Reveal.app/Contents/SharedSupport/iOS-Libraries/libReveal.dylib", 0x2);
command alias reveal_load_dev expr (void*)dlopen([(NSString*)[(NSBundle*)[NSBundle mainBundle] pathForResource:@"libReveal" ofType:@"dylib"] cStringUsingEncoding:0x4], 0x2);
command alias reveal_start expr (void)[(NSNotificationCenter*)[NSNotificationCenter defaultCenter] postNotificationName:@"IBARevealRequestStart" object:nil];
command alias reveal_stop expr (void)[(NSNotificationCenter*)[NSNotificationCenter defaultCenter] postNotificationName:@"IBARevealRequestStop" object:nil];
  • reveal_load_sim 是指的在模拟器加载Reveal，最好确保你Reveal的安装位置和上面的写得一样。
  • reveal_load_dev 是指的在真机调试或者模拟器下打开Reveal，不过如果实在真机，确保在libReveal.dylib放在了Copy Resources里，而非Link Binary with Libraries。
  • reveal_start 发送调试开始的通知
  • reveal_stop 发送终止的通知

然后在LLDB Console里头像bash那样load文件 ~/.the_reveal_load_file 就好啦！
