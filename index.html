关于苹果推送服务的证书

关于苹果推送服务的证书

之前的证书一直不是我弄的，所以浪费了好几个小时配合后端整推送的证书，网上的Demo其实很多很简单，但是也很杂，主要的有以下几个：

Mac Demo

最简单的要属于著名的PushMeBaby，这个证书是用的.cer的证书，从Apple Develper Portal直接下下来的证书就是，不过前提是这个证书是你的电脑请求的或者别人分享给你了私钥，程序运行的时候也能发现它会弹出访问Keychain的提示，但是这个不是生产环境常用的情况。

PHP Demo

另外一个Demo是PHP的，这个也是大家常用的一个版本，在RayWenderlich说得很详细，它的key是pem格式的证书文件。

生成这个pem，你需要得到从Developer Portal获得aps_development.cer以后，导入到Keychain，而后从Keychain导出aps_development.p12文件，导出以后，用openssl命令生成证书的pem文件：

$ openssl x509 -in aps_development.cer -inform der -out PushCert.pem

再将aps_development.p12生成私钥，在这个过程中，你需要输入私钥的密码，并且两次输入新的pem私钥的新密码：

$ openssl pkcs12 -nocerts -out PushChatKey.pem -in PushKey.p12
Enter Import Password:
MAC verified OK
Enter PEM pass phrase:
Verifying - Enter PEM pass phrase:

最后，生成一个pem文件：

$ cat PushCert.pem PushKey.pem > ck.pem

而后你就可以用这段PHP代码来直接测试push啦：

<?php

// Put your device token here (without spaces):
$deviceToken = 'aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa';

// Put your private key's passphrase here:
$passphrase = 'xxxxxx';

// Put your alert message here:
$message = 'My first push notification!';

////////////////////////////////////////////////////////////////////////////////

$ctx = stream_context_create();
stream_context_set_option($ctx, 'ssl', 'local_cert', 'ck.pem');
stream_context_set_option($ctx, 'ssl', 'passphrase', $passphrase);

// Open a connection to the APNS server
$fp = stream_socket_client(
    'ssl://gateway.push.apple.com:2195', $err,
    $errstr, 60, STREAM_CLIENT_CONNECT|STREAM_CLIENT_PERSISTENT, $ctx);

if (!$fp)
    exit("Failed to connect: $err $errstr" . PHP_EOL);

echo 'Connected to APNS' . PHP_EOL;

// Create the payload body
$body['aps'] = array(
    'alert' => $message,
    'sound' => 'default'
    );

// Encode the payload as JSON
$payload = json_encode($body);

// Build the binary notification
$msg = chr(0) . pack('n', 32) . pack('H*', $deviceToken) . pack('n', strlen($payload)) . $payload;

// Send it to the server
$result = fwrite($fp, $msg, strlen($msg));

if (!$result)
    echo 'Message not delivered' . PHP_EOL;
else
    echo 'Message successfully delivered' . PHP_EOL;

// Close the connection to the server
fclose($fp);
?>

Java Demo

在生产环境里用得最多的还是Java的版本Java APNS(https://github.com/notnoop/java-apns)吧，它需要的是一个.p12的私钥，我今天尝试了半天，发现直接导出的密钥是不能用的，在Google了一阵以后，尝试把PHP Demo里已经work的pem转成.p12，结果竟然推送成功。

具体步骤大概是：

在得到PushCert.pem以后PushKey.pem以后，输入如下：

$ chmod 0600 PushKey.pem
$ chmod 0644 PushCert.pem
$ openssl x509 -text -noout -in PushCert.pem
$ openssl pkcs12 -export -out certif.p12 -inkey Pushkey.pem -in Pushcert.pem

最后输入并且设置密码就好，成功！

Optional of Swift

Optional of Swift

“What is Optional” and “What can I do with it” may be your first two questions when it comes to Optional.

A question mark(?) or exclamatory mark(!) following a variable is the way Swift codes express Optional.

if let roomCount = john.residence?.numberOfRooms {
    println("John's residence has \(roomCount) room(s).")
} else {
    println("Unable to retrieve the number of rooms.")
}
// prints "Unable to retrieve the number of rooms."

As you may have known, in Swift compiler can speculate the type of a variable so most of the time you don’t need to assign the type whether it is a Bool, Int, Struct or String. While there always some ambiguity in Swift code compiler causing problem in compiler time or runtime, we have to help the compiler a little bit with a question mark(?) and exclamatory mark(!). We call this pattern Optional, a little cost for a variable coming without type.

用Jekyll与Github.io来搭建个人博客

用Jekyll与Github.io来搭建个人博客

下班完折腾折腾，九点钟边看着一百七十多分钟的《音乐之声》边开始了博客的迁移，现在真的没有比Jekyl+github.io更方便的了吧，Wordpress从来没有觉得顺畅过，当然，最关键的是github.io不需要和Linux打交道，纯静态页面。

现在开始：

没有gem的先安装gem，gem怎么安装的在这里。

三条命令你就能看到你本地的Jekyllrb了：

~ $ gem install jekyll
~ $ jekyll new myblog
~ $ cd myblog
~/myblog $ jekyll serve
# => Now browse to http://localhost:4000

在你自定义的myblog文件夹里，目录应该差不多类似下面：

├── _config.yml
├── _drafts
|   ├── begin-with-the-crazy-ideas.textile
|   └── on-simplicity-in-technology.markdown
├── _includes
|   ├── footer.html
|   └── header.html
├── _layouts
|   ├── default.html
|   └── post.html
├── _posts
|   ├── 2007-10-29-why-every-programmer-should-play-nethack.textile
|   └── 2009-04-26-barcamp-boston-4-roundup.textile
├── _data
|   └── members.yml
├── _site
└── index.html

这些文件与文件夹分别代表的是：

文件/文件夹 内容
_config.yml 包含所有的配置数据，许多选项都可以执行命令行来设置，不过在这个文件里设置能让你少记一些东西
_drafts 文件夹里包含还没有发布的博文，这些文件的名字一般都没有包含日期：title.MARKUP
_includes 包含页面共用的头首侧边栏什么的
_layouts 博文页面的模板
_posts 这些就是你的博文啦
_data 对于Ruby来说，一些固定格式的网站数据可以放到这，jekyll引擎将会自动加载这个文件夹里的.yml和.yaml文件，如果文件名是members.yml，那么你可以通过site.data.members来访问这个数据
_site 这个是生成的站点文件防止的地方，这个文件夹刚好也可以是你的git文件夹
index.html或者其他 都会被Jekyll转换
其他文件与文件夹 其他的一些CSS文件或者图片文件都会直接复制过去。

另外在配置文件里还有很多选项可以设置，不过暂时还没有关心的必要。

接下来是比较关乎日常写博客的内容，我们从Front-matter开始，任何一个有如下YAML Front-matter头格式的文件，将会被Jekyll做特殊处理: --- layout: post title: Blogging Like a Hacker ---

以下是你会常用到的Front-matter选项：

变量 描述
layout 就是你在_layout里面有设置的排版，自带的有default, post, page
permalink 如果你不希望博文的链接是/year/month/day/title.html/你可以主动设置博文的永久链接
published 如果你在生成站点的时候不想发表这篇文章，你可以把这个属性设置成false
category categories 相当于文章的tag、分类或者文件夹
tags 其实都差不多

NSFilemanager的线程安全性

如官方文档所说：

The methods of the shared NSFileManager object can be called from multiple threads safely. However, if you use a delegate to receive notifications about the status of move, copy, remove, and link operations, you should create a unique instance of the file manager object, assign your delegate to that object, and use that file manager to initiate your operations.

In iOS and Mac OS X v 10.5 and later you should consider using [[NSFileManager alloc] init] rather than the singleton method defaultManager. Instances ofNSFileManager are considered thread-safe when created using [[NSFileManager alloc] init].

[NSFileManager defautManager]不是线程安全的，但是[[NSFileManager alloc] init]是线程安全的，为了安全起见，不如全部用后者。

PS：这和NSManagedObjectContext其实是差不多的。

Objective-C里@public、@protected和@private更深一点点的区别

感觉其实很多语法书里也不会讲得那么详尽，或者大家自然而然的以为这三个属性和C++里的无甚区别，其实不然。

首先，一致的是，对于拥有这些属性变量的类TestClass：

@public 表示任何其他类可以访问这个变量。

@protected 表示只有子类可以访问这个变量。

@private 表示任何其他类不能访问这个变量。

如果不标注，默认所有变量都是@protected。

如果你对C++也不怎么熟悉（比如我），初看，你会觉得@public和@protected没有什么区别，因为一般在ObjC里，如果须要设置外部类要访问的变量，我们习惯把变量直接用@property和@synthesize直接生成get和set方法，但是其实ObjC还可以用指针->访问这些变量，这些语法可能延续自Smalltalk的习惯（待查证，不过注意Smalltalk可比C++老十几年，这是乔布斯从Xerox PARC偷来的两样东西之一，另一个东西是图形界面）。

例如以下代码，你可以尝试写入Xcode，不用编译，你就能意识到区别。

@interface TestClass : NSObject{
    @protected
    int protectedNumber;
    @private
    int privateNumber;
    @public
    int publicNumber;
}

@end

@implementation TestClass
@end

@interface TestChildClass : TestClass
@end

@implementation TestChildClass

- (instancetype)init{
    if(self = [super init]){
        self->protectedNumber = 1;
    }
    return nil;
}
@end

@interface OtherClass : NSObject

@end

@implementation OtherClass

- (void)testMethod{

    TestChildClass *testInstance = [TestChildClass new];
    testInstance->publicNumber = 1;
    NSLog(@"test number value is:%d",testInstance->publicNumber);
}

@end

另外，如果你熟悉ObjC的runtime编程，访问过isa的struct，你应该早就明白了。
